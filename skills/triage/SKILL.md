# /triage — the issue state machine

> **Engine:** balanced — Claude `sonnet-5` (medium) · OpenAI `gpt-5.6-terra` (medium) · Subagent: NO — the owner directs transitions.
>
> **Concurrency:** Interactive: present buckets, WAIT for the owner's pick; one issue at a time; repro attempts may run in a subagent, then rejoin.

Untriaged issues are noise; /resume can only route work that has been evaluated. Triage moves every issue to exactly one category (`bug` | `enhancement`) and one state (`needs-triage` | `needs-info` | `ready-for-agent` | `ready-for-human` | `wontfix`). Every comment posted during triage starts with: `> *This was generated by AI during triage.*`

## Transitions

Unlabeled → `needs-triage` → one of `needs-info` / `ready-for-agent` / `ready-for-human` / `wontfix`. `needs-info` returns to `needs-triage` when the reporter replies. The owner can override any transition — confirm, then apply; flag anything unusual.

## Show what needs attention

Present three buckets, oldest first, with counts and a one-line summary each: (1) unlabeled, (2) `needs-triage`, (3) `needs-info` with reporter activity since the last triage notes. Let the owner pick.

## Triage one issue

1. **Gather.** Read the full issue and prior triage notes (never re-ask resolved questions). Explore the codebase using CONTEXT.md vocabulary; respect ADRs in the area; surface any prior `wontfix` that resembles this issue.
2. **Recommend.** State category + state with reasoning and a brief relevant-code summary. Wait for direction.
3. **Reproduce (bugs).** Attempt repro before any grilling: reporter's steps, code trace, tests. Report repro-with-code-path, failed repro, or insufficient detail (a strong `needs-info` signal).
4. **Grill if needed.** Run /grill to flesh out an underspecified issue.
5. **Apply.** `ready-for-agent` → post an agent brief (goal, constraints, ACs, repro, files). `ready-for-human` → same brief plus why it can't be delegated. `needs-info` → triage notes: what's established so far, and specific actionable questions for @reporter. `wontfix` → polite explanation, record the rationale durably, close.
